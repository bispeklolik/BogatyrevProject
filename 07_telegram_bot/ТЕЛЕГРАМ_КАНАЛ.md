## Памятка по оформлению постов
- Без эмодзи и хэштегов.
- Жирным — только смысловые повороты (1–2 на пост).
- Разделитель смысловых блоков: `·  ·  ·`
- Цитаты: «текст» — Автор.
- Длина: эссе 1500–3000 знаков (одна мысль через 3–5 абзацев); реплика 300–500 (одна точная мысль).
- Одна мысль на пост. Первая строка цепляет мыслью, а не кликбейтом.
- **Юмор — твоё.** Ирония, стёб, сарказм. Цель стёба — социум, инфоцыгане, попса; не читатель.
- Нельзя: «друзья/ребята/дорогие подписчики», «ставь лайк/огонёк», обратные таймеры, «измени жизнь за неделю», восклицательные знаки пачками.
- Ссылка на сайт — в описании канала; в постах изредка и по делу.

## ТЗ для разработчика: умный поиск по смыслу (semantic search)

**Цель.** Пользователь пишет боту тему/вопрос обычными словами — бот возвращает 3–5 постов канала, релевантных **по смыслу**, со ссылками и короткой строкой «почему подходит». Опционально — формулирует короткий ответ в тоне Матвея со ссылками на посты.

**Как работает (RAG):**
1. **Индексация.** Каждый пост канала → текст + ссылка (`t.me/<канал>/<id>`) + дата → «эмбеддинг» (числовой смысловой вектор) → векторная база.
2. **Обновление.** Бот-админ канала ловит каждый новый пост (`channel_post`) и добавляет его эмбеддинг автоматически. Старые посты — разовый импорт истории.
3. **Поиск.** Запрос пользователя → эмбеддинг → поиск ближайших по косинусной близости → выдача ссылок + «почему».
4. **Ответ (опц.).** LLM берёт найденные посты и пишет короткий ответ в тоне Матвея, ссылаясь на них. Без выдумывания: если релевантного нет — честно «по этой теме пока не писал».

**Стек:**
- Бот: Python + aiogram (тот же, что «Спутник практики»).
- База: PostgreSQL + расширение **pgvector** (уже в плане бота).
- Эмбеддинги: `text-embedding-3-small` (OpenAI) / локальная `e5-multilingual` / YandexGPT Embeddings — выбрать по доступности и цене в РФ.
- LLM для ответа (опц.): Claude / GPT / YandexGPT.

**Интерфейс в боте:** команда `/поиск` или режим «спроси о теме» → пользователь шлёт текст → выдача.

**Стоимость:** эмбеддинги дёшевы (центы за тысячи постов); LLM-ответ — по факту запроса.

**Бонус:** тот же индекс можно подключить к сайту (поиск по эссе) — отдельная страница или виджет.

**Куда встроить:** новый модуль бота «Спутник практики» (см. `bot.html`). Добавить как 7-ю функцию.

> Отстройка: у психологов в Telegram семантического поиска по постам нет. Это заметная и аутентичная методу фича.

## ТЗ для разработчика: запись через Telegram (бронирование)

**Принцип.** Основной и надёжный канал записи — форма на сайте (работает даже при блокировках Telegram). Запись в боте — удобная добавка; на каждом шаге доступна кнопка **«Записаться на сайте»** (`matveybogatyrev.ru#booking`) как фолбэк.

**Поток:**
1. **Старт** — кнопка «Записаться» (в меню бота и по ссылке из «Навигатора» канала).
2. **Выбор формата** (inline-кнопки):
   - *Глубинный разбор · 12 000 ₽* → полный поток ниже.
   - *Сопровождение / Великое делание* → только заявка-обсуждение: в один клик не продаются, идут после разбора/знакомства, оплата по договорённости.
   - *«Не уверен — нужна рекомендация»* → заявка с пометкой.
3. **Мини-заявка** (как на сайте, короче): запрос своими словами; возраст; формат. Контакт берётся из Telegram-аккаунта.
4. **Ответ бота:** «Заявка у Матвея. Отвечу за 1–2 рабочих дня — подходим ли мы друг другу». (Матвей сначала смотрит — принцип «я выбираю, с кем работать».)
5. **Подтверждение (Матвей в админке):** match / no-match. При match открывает 2–3 свободных окна.
6. **Выбор времени:** бот показывает окна, открытые Матвеем → человек выбирает.
7. **Оплата:**
   - **Сейчас (MVP, без самозанятого):** перевод по номеру телефона (СБП на карту) + ручное подтверждение. Бот присылает сумму и реквизиты, кнопка «Я оплатил» → Матвей подтверждает в админке. Чека на этом этапе нет.
   - **Позже (после самозанятого):** оплата прямо в боте через ЮKassa / Telegram Payments; чек НПД формируется автоматически.
8. **После оплаты:** бот запускает анкету первой встречи (функция 01) и напоминает за 24 часа.

**Админка (для Матвея):** очередь заявок, кнопки match/no-match, открытие окон, подтверждение оплаты, просмотр анкет.

**Фолбэк:** на шагах 1–6 всегда видна кнопка «Записаться на сайте» (на случай недоступности Telegram).

**Юридически (важно):** приём оплаты без статуса = незадекларированный доход с налоговым риском, особенно вместе с публичной рекламой под своим именем. Рекомендация: оформить самозанятого **до** запуска Яндекс.Директа и оплаты в боте — тогда неформальный период совпадает с тихим до-рекламным стартом, а окно риска минимально.

**Куда встроить:** расширение функции 06 бота «Спутник практики» (см. `bot.html`).
