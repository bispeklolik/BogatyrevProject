# Бот «Спутник практики» — сводное ТЗ для разработчика

Единый документ для передачи разработчику. Полный концепт и UX-сценарии — в `bot.html`. Новые функции — в `ТЕЛЕГРАМ_КАНАЛ.md` (умный поиск, запись). Здесь — **приоритеты сборки, правки к концепту и план по фазам**.

---

## 0. Правки к исходному концепту (важно, перекрывают bot.html)
- **Город:** Санкт-Петербург (не Москва). Очно — СПб; Москва и остальная РФ/СНГ — онлайн.
- **Оплата:** на старте — перевод по номеру телефона (СБП) + ручное подтверждение Матвеем (без чека). Позже (после регистрации самозанятого) — ЮKassa / Telegram Payments с авто-чеком НПД.
- **Эзотерика — не отсекаем.** Юнгианские/символические механики (сны, архетипы, тень) уместны; тон — глубоко, но без догм и «сонников».
- **Юмор:** тексты бота допускают сухую иронию в духе Матвея (но без эмодзи).
- **Telegram может блокироваться в РФ:** бот — не единственный путь. Всё критичное (заявка) дублируется на сайт; на каждом шаге — фолбэк-ссылка `matveybogatyrev.ru`.
- **Анти-геймификация** (как в концепте): без стриков, очков, бейджей, «вернись, мы скучаем».

## 2. Phase 1 (MVP) — что именно собрать

**Онбординг `/start`:** короткое приветствие в тоне Матвея + меню: «Записаться» · «Задать вопрос» · «О форматах». (Утренний вопрос — заглушка «скоро».)

**Функция 01 · Анкета первой встречи.** Запускается после подтверждённой записи. 5 вопросов (биография-минимум, запрос своими словами, что пробовал и почему не сработало, чего НЕ хочешь от встречи, что должно произойти). Ответы сохраняются, отправляются Матвею за 24 ч до встречи.

**Функция 02 · «Один вопрос» по четвергам.** Пользователь задаёт один развёрнутый вопрос раз в 3 месяца → очередь → Матвей отвечает по четвергам через админку → бот доставляет ответ. **Доп.: флаг «опубликовать анонимно»** — с согласия ответ идёт в контент канала (связь с рубрикой вовлечения).

**Запись (расширение функции 06).** Полный поток — см. `ТЕЛЕГРАМ_КАНАЛ.md` → «ТЗ: запись через Telegram». Кратко: формат → мини-заявка → «Матвей ответит за 1–2 дня» → match/no-match в админке → выбор окна → оплата (перевод+подтверждение на MVP) → анкета (фн 01) → напоминание. Сопровождение/«Великое делание» — только заявка-обсуждение, без оплаты в один клик.

**Админка (web, для Матвея):** очередь заявок (match/decline, открыть окна, подтвердить оплату), очередь вопросов (ответить, пометить «публиковать»), просмотр анкет.

**Везде фолбэк** на запись через сайт (на случай недоступности Telegram).

## 3. Стек
- **Бот:** Python + aiogram.
- **БД:** PostgreSQL (+ расширение **pgvector** для Phase 2 — поиск по смыслу). Для MVP достаточно базовых таблиц.
- **Планировщик:** APScheduler (четверговые ответы, напоминания, позже — утренние вопросы).
- **Оплата:** MVP — без интеграции (перевод + ручное подтверждение); позже — ЮKassa / Telegram Payments.
- **Хостинг:** VPS (Selectel / Timeweb), ~200–400 ₽/мес на старте.
- **Админка:** простой web-UI (FastAPI / Flask).
- **Phase 2+:** эмбеддинги (OpenAI / e5-multilingual / YandexGPT) для поиска; LLM (Claude/GPT/YandexGPT) для AI-дневника и ответов поиска.

**Набросок данных (MVP):** `users` (tg_id, username, consent, tz) · `requests` (формат, текст, возраст, статус, окно) · `questions` (текст, статус, ответ, public) · `payments` (метод, сумма, статус) · `анкеты`. На Phase 2 добавится `posts_index` (текст поста, ссылка, embedding).

## 4. Этика AI-функций (с Phase 3, обязательно)
Из концепта, сохраняем строго: AI **не диагностирует**, **не даёт ответов** (только наблюдения и вопросы), **не имитирует эмпатию**, **не заменяет психолога**; данные анонимизируются при отправке в API; пользователь может удалить любую запись/весь архив; в шапке ответа AI всегда «Это разбор AI».
