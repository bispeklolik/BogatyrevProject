# Как редактировать тексты в файлах вручную

Все артефакты — обычные HTML-файлы. Их можно открыть в любом текстовом редакторе (VS Code, Sublime Text, Notepad++, даже Блокнот) и менять текст без знания HTML/JS. Главное — понимать, **где НЕ надо ничего трогать**.

---

## Что можно трогать

### 1. Текст внутри тегов

Это самое простое. Текст между `<тег>` и `</тег>` — это то, что видит пользователь.

```html
<h1>Поле, в которое заходит <em>осмысленная практика.</em></h1>
```

Здесь `Поле, в которое заходит осмысленная практика.` — текст, можно менять. `<h1>`, `</h1>`, `<em>`, `</em>` — теги, не трогать.

### 2. Текст в кавычках в JS-данных

В файле `filter.html` есть массивы данных, например:

```js
const LIKERT = [
  {id:'a1', text:'Я чувствовал(а) нервозность, тревогу или напряжённость', scale:'anxiety'},
  ...
];
```

Можно менять текст в одинарных кавычках после `text:`. Не трогай: `id:'a1'`, `scale:'anxiety'`, кавычки, запятые, скобки.

### 3. Цены, длительности, заголовки секций

Просто ищешь нужное число или фразу, меняешь.

---

## Что трогать НЕ надо

### 1. Любой код в `<style>...</style>`
Это CSS — оформление. Если поменять, может «съехать» вёрстка.

### 2. Любой код в `<script>...</script>`
Это JavaScript — логика. Сломаешь — перестанут работать чекбоксы, фильтры, графики.

**Исключение:** текстовые литералы в JS (текст внутри `'одинарных'` или `"двойных"` кавычек) можно менять, но осторожно — не трогая сами кавычки.

### 3. Атрибуты тегов
Например, `<div class="kpi">` — `class="kpi"` не трогать, это связано с оформлением.
`<a href="https://...">` — `href` не трогать (это ссылка), а текст ссылки между `<a>` и `</a>` — можно.

### 4. Спецсимволы
- `&nbsp;` — неразрывный пробел
- `&mdash;` или `—` — длинное тире
- `&laquo;`, `&raquo;` или `«»` — кавычки
- `<br/>` — перенос строки

Их можно копировать, но ломать структуру (`&nbsp` без `;` — поломка).

## Совет по безопасности

Перед тем как править — **сделай копию файла**. Например, `filter.html` → `filter_backup.html`. Если что-то сломается, всегда можно вернуть.

Чтобы проверить, не сломался ли файл после правок — открой его в браузере двойным кликом. Если открывается и работает — всё в порядке.
